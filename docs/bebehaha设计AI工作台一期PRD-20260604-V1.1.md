版本：V1.1

## 1. 项目定位

本项目一期交付一个面向 bebehaha 设计团队的 AI 设计工作台。

一期目标不是交付完整企业 AI 平台，也不是单纯提供 AI 生图工具，而是在设计项目中完成三件事：

1. 让设计师以项目为单位使用 AI 生图、改图和沉淀结果。
2. 让老板能够查看项目、成员、素材和用量情况。
3. 将 Input 图、Output 图、Prompt、对话记录和项目过程沉淀为企业设计资产。

## 2. 首期账号与角色

首期账号配置：

| 角色 | 数量 | 说明 |
| --- | ---: | --- |
| 管理员 / 老板 | 1 | 曹总，拥有公司级查看权限 |
| 设计师 / 员工 | 3 | 设计执行人员，使用项目画板和 AI 设计助手 |

首期不支持新增管理员，不在界面中提供复杂权限变更、停用/启用账号等管理能力。

新增账号、权限扩展、管理员扩展等需求，后续另行确认。

## 3. 角色权限总览

| 模块 | 管理员 / 老板 | 设计师 / 员工 |
| --- | --- | --- |
| 工作台首页 | 可查看团队项目、成员活跃、生成用量 | 不可访问，登录后进入我的项目 |
| 项目管理 | 查看全部项目，可归档/恢复 | 查看自己创建或参与的项目，可归档/恢复 |
| 项目画板 | 只读查看，可下载、归档/恢复 | 可编辑、上传、生成、对话、下载、归档/恢复 |
| 素材库 | 查看团队全部素材 | 查看自己创建或参与项目的素材 |
| 成员管理 | 查看成员、参与项目和使用情况 | 不可访问 |
| 账户用量 | 查看积分余额、充值入口、项目升级入口、团队用量和模型调用明细 | 不可访问账户用量页 |

一期包含 6 个核心页面：

1. 工作台首页

# 页面 1：工作台首页

## 页面定位

管理员登录后的默认首页，用于快速查看团队设计 AI 工作台的整体使用情况。

## 使用角色

管理员 / 老板。

设计师不访问该页面。

## 核心目标

- 查看当前项目推进情况。
- 查看团队近期是否在使用 AI 工作台。
- 查看每个设计师活跃情况。
- 查看本月整体生成用量。

## 页面结构

- 顶部：系统名称、当前用户、团队名称。
- 左侧导航：工作台、全部项目、团队素材库、成员管理、账户用量。
- 主内容：数据概览、最近项目、成员活跃情况、近期生成趋势。

## 数据概览

| 卡片 | 说明 |
| --- | --- |
| 进行中项目数 | 当前状态为进行中的项目数量 |
| 已归档项目数 | 已归档项目数量 |
| 本月生成次数 | 本月团队 AI 图片生成次数 |
| 活跃设计师人数 | 本月有使用记录的设计师数量 |

首页不展示费用金额。

## 最近项目列表

字段：

- 项目名称
- 负责人
- 协作成员
- 状态
- 生成次数
- 最近更新时间
- 操作：查看项目

默认按最近更新时间倒序展示。

## 成员活跃情况

字段：

- 成员姓名
- 参与项目数
- 本月生成次数
- 最近活跃时间
- 状态：活跃 / 低活跃 / 暂无使用

状态规则：

- 活跃：近 7 天有使用记录。
- 低活跃：近 30 天有使用记录，但近 7 天无使用。
- 暂无使用：本月无使用记录。

## 权限规则

管理员可访问该页面并查看全团队数据。

设计师不显示工作台入口，登录后进入我的项目。

## 验收标准

管理员登录后可看到：

- 进行中项目数
- 已归档项目数
- 本月生成次数
- 活跃设计师人数
- 最近项目列表
- 成员活跃情况
- 最近 7 天生成趋势

# 页面 2：项目管理页

## 页面名称

管理员端：全部项目  
员工端：我的项目

## 页面定位

用于查看、创建、进入、归档和恢复设计项目。

每个 Project 对应一个具体设计任务，项目内包含画板、素材、Prompt、对话和生成记录。

## 使用角色

管理员、设计师。

## 核心目标

- 查看有权限的项目。
- 新建设计项目。
- 邀请协作成员。
- 进入项目画板。
- 归档项目。
- 恢复已归档项目。

## 项目列表字段

| 字段 | 说明 |
| --- | --- |
| 项目名称 | Project 名称 |
| 项目说明 | 简短说明，可选 |
| 负责人 | 项目主要负责人 |
| 协作成员 | 被邀请参与项目的成员 |
| 状态 | 进行中 / 已归档 |
| 素材数 | 项目内 Input 图 + Output 图数量 |
| 生成次数 | 项目内 AI 生成次数 |
| 最近更新时间 | 最近一次项目操作时间 |
| 操作 | 进入画板 / 查看归档 / 归档 / 恢复 |

## 新建项目

字段：

| 字段 | 是否必填 | 说明 |
| --- | --- | --- |
| 项目名称 | 是 | 例如：儿童保温杯详情页设计 |
| 项目说明 | 否 | 简述本次设计目标 |
| 负责人 | 是 | 默认当前创建人 |
| 协作成员 | 否 | 可邀请团队内设计师 |
| 参考素材 | 否 | 初始 Input 图 |
| 初始 Prompt | 否 | 初始生成需求 |

项目类型暂不预设。

## 归档与恢复

归档后：

- 项目状态变为已归档。
- 项目数据、素材、Prompt、对话和生成记录保留。
- 默认只读。
- 必须恢复为进行中后才能继续编辑或生成。

管理员可归档/恢复任意项目。  
设计师可归档/恢复自己创建、负责或参与的项目。

## 权限规则

管理员：

- 查看全部项目。
- 进入任意项目。
- 归档/恢复任意项目。
- 按成员、状态筛选。

设计师：

- 查看自己创建或被邀请参与的项目。
- 可新建项目。
- 可邀请其他设计师加入项目。
- 可归档/恢复自己有权限的项目。

## 验收标准

系统必须记录：

- 项目创建人
- 负责人
- 协作成员
- 创建时间
- 最近更新时间
- 项目状态
- 素材数
- 生成次数

# 页面 3：项目画板页

## 页面定位

围绕单个项目展开 AI 设计工作，以画布形式组织参考图、Prompt、生成结果和修改反馈，并通过右侧 AI 设计助手辅助调整 Prompt 和生成图片。

## 使用角色

管理员、项目负责人、设计师、项目协作成员。

## 页面结构

- 顶部：项目信息、状态、负责人、协作成员、返回、下载、归档/恢复。
- 左侧：项目素材区。
- 中间：画布工作流区。
- 右侧：AI 设计助手。

## 画布节点

| 节点类型 | 说明 |
| --- | --- |
| Input 图片节点 | 用户上传的参考图、产品图、场景图 |
| Prompt 节点 | 用于驱动模型生成的提示词 |
| Output 图片节点 | AI 生成结果图 |
| 反馈 / 备注节点 | 设计师对结果图的修改意见、版本说明 |

节点之间用箭头连接，用于表达工作流关系：

Input 图 -> Prompt 节点 -> Output 图 -> 反馈节点 -> 新 Prompt -> 新 Output 图

一期只表达画布组织和过程沉淀，不承诺复杂自动编排。

## 左侧素材区

Tab：

- 全部
- Input 图
- Output 图

素材卡片字段：

- 缩略图
- 类型：Input / Output
- 来源：上传 / AI 生成
- 创建人
- 创建时间
- 操作：添加到画布 / 下载 / 继续修改

## 右侧 AI 设计助手

右侧是对话式 Agent，不是单纯 Prompt 输入框。

能力：

- 支持选择当前画布节点作为上下文。
- 支持自然语言反馈。
- 支持优化当前 Prompt。
- 支持基于选中图片继续改图。
- 支持生成更多版本。
- 支持保留对话记录。

示例：

用户：这个图产品不够突出，背景太杂。  
AI：建议将 Prompt 调整为更干净的电商主图风格，并减少背景干扰。  
操作：应用到 Prompt 节点 / 直接生成。

## Output 生成规则

生成结果自动进入两个地方：

1. 当前项目画布，作为新的 Output 节点。
2. Output 素材库。

系统记录：

- 操作人
- 时间
- 模型
- Prompt
- Input 图
- Output 图
- 所属项目
- Input / Output 关系

## 权限规则

管理员 / 老板：

- 只读查看。
- 可查看画布、素材、Prompt、对话记录、生成记录。
- 可下载素材。
- 可归档/恢复项目。
- 不可编辑画布。
- 不可生成图片。
- 不可修改 Prompt。

设计师：

- 可上传 Input 图。
- 可新增 Prompt 节点。
- 可选择模型生成图片。
- 可通过 AI 设计助手调整 Prompt。
- 可基于 Output 图继续改图。
- 可新增反馈 / 备注。
- 可拖动画布节点。
- 可下载素材。
- 可归档/恢复项目。

已归档项目：

- 只读。
- 不可上传、生成、编辑。
- 必须恢复为进行中后才能继续操作。

## 验收标准

设计师必须能完成：

- 上传参考图。
- 输入或调整 Prompt。
- 选择模型。
- 发起生成。
- 查看生成结果。
- 保存生成结果到画布和素材库。
- 查看生成记录。
- 下载生成图片。

管理员必须能完成：

- 查看任意项目画板。
- 查看素材、Prompt、对话和生成记录。
- 下载素材。

# 页面 4：素材库页

## 页面名称

管理员端：团队素材库  
员工端：我的素材

## 页面定位

集中管理项目中产生的 Input 图和 Output 图，支持按项目、成员、类型、时间、模型筛选，并查看素材来源记录和单张下载。

## 素材类型

| 类型 | 说明 |
| --- | --- |
| Input 图 | 用户上传的参考图、产品图、场景图 |
| Output 图 | AI 模型生成的结果图 |

## 展示方式

网格卡片 + 右侧详情抽屉。

素材卡片字段：

- 图片缩略图
- 素材类型
- 所属项目
- 创建人
- 创建时间
- 模型，Output 图显示
- 操作：详情 / 下载 / 进入项目

## 素材详情

Output 图详情必须展示：

- 所属项目
- 创建人
- 创建时间
- 使用模型
- 完整 Prompt
- 关联 Input 图
- 生成记录
- 下载
- 进入项目
- 继续修改，设计师可见

## 权限规则

管理员：

- 查看团队全部素材。
- 按项目、成员、类型、时间、模型筛选。
- 查看完整 Prompt 和来源记录。
- 下载任意素材。
- 进入任意来源项目。
- 不在素材库直接编辑图片。

设计师：

- 查看自己创建或参与项目中的素材。
- 可查看同项目其他成员生成的素材。
- 可查看完整 Prompt 和来源记录。
- 可下载有权限素材。
- 可进入有权限来源项目。
- 可对 Output 图继续修改。

## 边界

- 一期不支持批量下载。
- 一期不做素材删除。
- 项目归档后，素材仍可查看和下载。

## 验收标准

管理员和设计师必须能区分 Input 图 / Output 图，并查看 Output 图的完整 Prompt 和来源记录。

# 页面 6：账户用量页

## 页面定位

用于管理员查看积分余额、充值入口、项目升级入口，以及团队 AI 使用情况，包括生成次数、成员使用情况、项目使用情况和模型调用记录。

该页面是老板账户级能力页面，统一呈现积分与用量，不向设计师开放。

## 使用角色

管理员 / 老板。

设计师不访问账户用量页。

## 页面模块

1. 积分余额
2. 充值入口
3. 项目升级入口
4. 用量概览
5. 成员用量排行
6. 项目用量排行
7. 模型调用明细

## 积分余额

展示：

- 当前积分余额
- 本月已使用积分

说明：

积分用于模型调用、图片生成等 AI 能力消耗。

## 充值入口

管理员可点击「充值积分」进入平台充值流程。

该能力由平台完整充值能力支持，不使用联系充值弹窗。

## 项目升级入口

管理员可点击「项目升级」申请下一阶段扩展。

点击后弹出提示：

> 请联系18682302338进行扩展升级

## 用量概览

展示：

- 本月生成次数
- 活跃成员数
- 使用项目数
- 主要使用模型

## 筛选

- 时间
- 项目
- 成员
- 模型

## 模型调用明细

字段：

- 时间
- 成员
- 项目
- 模型
- 操作类型：文生图 / 图生图 / 改图
- Prompt 摘要
- 结果
- Output 图
- 消耗积分

积分规则：

- 首页不展示费用金额。
- 账户用量页展示积分余额与已使用积分。
- 明细中展示单次消耗积分。
- 模型调用、图片生成等 AI 能力按积分消耗记录。

## 边界

- 一期不做用量报表导出。
- 不做复杂财务结算。

## 验收标准

管理员可查看积分余额、本月已使用积分，使用充值入口，使用项目升级入口，查看团队生成次数、成员用量、项目用量、模型调用明细，并能从明细进入对应项目或素材。

设计师不显示账户用量入口。

## 5. 一期不包含项

为避免范围扩张，一期明确不包含：

- 完整企业 AI 平台交付。
- 复杂协作型 Agent 分析。
- 运营数据报表。
- 电商平台数据接入。
- 用量报表导出。
- 批量下载素材。
- 删除素材。
- 新增管理员。
- 前端复杂权限变更。
- 停用/启用账号。
- 源码买断。

## 6. 数据与知识产权边界建议

客户拥有其项目数据、上传素材、生成图片、Prompt 和使用记录的使用权与导出权。

本次交付为基于外脑.ai平台的 bebehaha 专属设计 AI 工作台使用环境。

外脑.ai保留底层平台、通用技术框架和产品能力的知识产权。

## 7. 商务边界建议

项目费用：6 万元未税。  
税费：按 6%另计。  
含税金额：63,600 元。

模型/API 调用消耗按积分记录，客户可通过平台充值入口进行积分充值。

如一期效果良好，后续可继续讨论二期功能建设，不在一期中写成固定续费。
